## Goal
Stop new structural drift while the migration is in progress.

## Current repo cues
- `backend/app/services/`
- `backend/app/utils/`
- `frontend/src/components/hooks/`
- `frontend/src/lib/`
- `frontend/src/hooks/`
- `frontend/src/contexts/`

## Scope
- Add or adopt a repo-root `STRUCTURE.md` that states the target layout and placement test.
- Update `AGENTS.md`, `CLAUDE.md`, and `CONTRIBUTING.md` so contributors know which roots are allowed and which legacy paths are frozen.
- Mark `backend/app/services/`, `backend/app/utils/`, and `frontend/src/components/hooks/` as frozen destinations for new files.
- Allow `frontend/src/hooks/`, `frontend/src/lib/`, and `frontend/src/contexts/`, but document their strict meanings.
- Add a lightweight structural check (script or CI step) that blocks new files under frozen paths except an explicit allowlist for temporary shims.
- Block creation of new vague bucket names such as `utils`, `helpers`, `common`, or `misc` under active domains unless explicitly allowlisted.

## Out of scope
- Moving existing files.
- Changing runtime behavior.
- Renaming modules beyond docs and guardrail tooling.

## Deliverables
- Updated repo docs with placement rules.
- A structural validation script and CI hook.
- An allowlist mechanism for temporary migration shims.

## Acceptance criteria
- New files under frozen legacy buckets fail the structural check unless allowlisted.
- The placement rules are discoverable from the repo root and contributor docs.
- No product behavior changes are introduced.

## Notes
This ticket is the gating ticket. Do it first or the rest of the migration will fight new drift every day.
