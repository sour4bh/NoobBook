## Goal
Create the target backend shape before moving behavior into it.

## Current repo cues
- `backend/app/api/`
- `backend/app/config/`
- `backend/app/services/`
- `backend/app/utils/`

## Scope
- Create backend target packages: `base/`, `platform/`, `auth/`, `projects/`, `sources/`, `chat/`, `studio/`, `settings/`, and `brand/`.
- Add `__init__.py` files and small package-level docs or module comments describing each package's ownership.
- Create temporary public-surface modules where needed so call sites can migrate gradually.
- Add temporary compatibility shims from old import paths to new public surfaces when necessary to avoid a flag day.
- Write an old-path to new-path mapping doc for contributors.

## Out of scope
- Large behavior refactors.
- Changing route contracts.
- Deleting old packages yet.

## Deliverables
- New backend package skeleton.
- Temporary compatibility shims.
- Path mapping doc.

## Acceptance criteria
- All target backend packages exist.
- Existing imports continue to resolve through shims or updated paths.
- No domain behavior is stranded without an obvious target package.

## Notes
This ticket creates the landing zones. Later tickets should move behavior into these packages rather than inventing new homes.
