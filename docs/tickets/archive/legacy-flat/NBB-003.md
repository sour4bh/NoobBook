## Goal
Give auth one home instead of splitting it across `services/auth` and `utils/auth_middleware.py`.

## Current repo cues
- `backend/app/services/auth/`
- `backend/app/utils/auth_middleware.py`
- `backend/app/api/auth/`

## Scope
- Move auth middleware, request identity resolution, RBAC, and permission helpers into `backend/app/auth/`.
- Create explicit public modules such as `middleware.py`, `identity.py`, `policy.py`, and `rbac.py`.
- Update route code so auth imports come from `backend/app/auth/` only.
- Keep route/framework adapters near the boundary; keep policy and identity logic in auth-owned modules.
- Add or update tests around import paths and basic auth wiring.

## Out of scope
- Changing RBAC semantics.
- Changing single-user vs multi-user behavior.
- Removing admin fallback behavior.

## Deliverables
- Auth package with clear public surfaces.
- Updated imports in routes and services.
- Basic regression tests for auth wiring.

## Acceptance criteria
- No auth logic remains in `backend/app/utils/` except temporary shims.
- Routes and services import auth from `backend/app/auth/` public surfaces.
- Auth ownership is obvious from the path layout.

## Notes
This is a structure ticket, not a policy rewrite ticket.
