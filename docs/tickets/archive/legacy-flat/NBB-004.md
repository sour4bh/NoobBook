## Goal
Make external systems and file/runtime adapters live under one explicit edge package.

## Current repo cues
- `backend/app/services/integrations/`
- `backend/app/utils/docx_utils.py`
- `backend/app/utils/pdf_utils.py`
- `backend/app/utils/pptx_utils.py`
- `backend/app/utils/presentation_export_utils.py`
- `backend/app/utils/screenshot_utils.py`

## Scope
- Move vendor-specific integrations from `services/integrations/` into `platform/` subpackages.
- Create explicit platform ownership by concern, e.g. `platform/llm/`, `platform/search/`, `platform/storage/`, `platform/voice/`, `platform/google/`, `platform/files/`, `platform/mcp/` as appropriate.
- Move generic file-format and runtime adapters into `platform/files/` or another explicit platform subpackage.
- Keep domain orchestration out of platform; platform should adapt external systems, not decide product policy.
- Update imports so external SDK usage is concentrated under `platform/`.

## Out of scope
- Changing prompts or product workflows.
- Behavioral rewrites of integrations.
- Studio or chat domain refactors beyond import moves.

## Deliverables
- Platform package with vendor/runtime subpackages.
- Updated imports from domains into platform.
- A clear rule that external SDKs enter through platform adapters.

## Acceptance criteria
- External SDK imports are concentrated under `backend/app/platform/`.
- File-format/runtime adapter code no longer lives in `backend/app/utils/`.
- No domain orchestration logic is introduced into platform modules.

## Notes
This ticket reduces hidden edges. It also makes later domain migrations easier because domains can depend on platform without importing vendor-specific code directly.
