## Goal
Turn chat into one owned domain with a small public surface and internal modules by responsibility.

## Current repo cues
- `backend/app/services/chat_services/`
- `backend/app/services/chat_services/main_chat_service.py`
- `backend/app/api/chats/`
- `backend/app/api/messages/`

## Scope
- Create `backend/app/chat/` with modules such as `service.py`, `orchestrator.py`, `prompt_builder.py`, `tool_registry.py`, `tool_router.py`, `streaming.py`, and `schemas.py` where appropriate.
- Move chat naming, orchestration, prompt assembly, and streaming mechanics under `chat/` instead of leaving them in generic service buckets.
- Reduce `main_chat_service.py` to a forwarding shim or thin facade during the migration.
- Define a clear public surface that other domains can import without reaching into chat internals.
- Keep permission checks and identity inputs explicit at the chat boundary.

## Out of scope
- Changing chat behavior or tool policy semantics.
- Replacing the Claude loop.
- Auth redesign.

## Deliverables
- Owned `backend/app/chat/` package.
- Thin legacy chat shim or deleted legacy chat service when safe.
- Updated imports and module boundaries.

## Acceptance criteria
- The main chat flow is understandable from `backend/app/chat/` alone.
- Legacy `main_chat_service.py` is reduced to a shim/facade or removed.
- Other domains import chat public surfaces rather than chat internals.

## Notes
Keep module boundaries deep. Avoid replacing one giant file with a dozen shallow wrappers.
