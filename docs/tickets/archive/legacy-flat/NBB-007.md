## Goal
Stop scattering each analysis feature across `ai_agents/`, `tool_executors/`, and `tools/`.

## Current repo cues
- `backend/app/services/ai_agents/csv_analyzer_agent.py`
- `backend/app/services/tool_executors/csv_analyzer_agent_executor.py`
- `backend/app/services/tool_executors/analysis_executor.py`
- `backend/app/services/tools/analysis_agent/`
- `backend/app/services/tool_executors/database_agent_executor.py`
- `backend/app/services/tool_executors/freshdesk_agent_executor.py`

## Scope
- Create `backend/app/sources/analysis/shared/`, `csv/`, `database/`, and `freshdesk/`.
- Move each analysis feature's agent/orchestration, executor/tool logic, tool specs, and schemas into the same slice.
- Move shared analysis-only code into `sources/analysis/shared/`.
- Keep file paths and naming consistent enough that one directory explains one feature end-to-end.
- Preserve current runtime behavior while changing ownership and import paths.

## Out of scope
- Replacing the free-form code execution contract.
- Redesigning analysis semantics.
- Security hardening beyond file ownership changes.

## Deliverables
- Owned analysis slices under `sources/analysis/`.
- Updated imports from routes/chat/tools into analysis public surfaces.
- Reduced cross-taxonomy scattering for CSV/database/Freshdesk analysis.

## Acceptance criteria
- Each structured analysis feature can be understood from one directory.
- Analysis-specific files are no longer split across three top-level taxonomies.
- Legacy analysis paths are reduced to temporary shims or removed.

## Notes
Keep this structural. The security and contract redesign work deserves separate tickets later.
