## Goal
Use blog generation as the template vertical slice for the rest of studio.

## Current repo cues
- `backend/app/api/studio/blogs.py`
- `backend/app/services/tool_executors/blog_agent_executor.py`
- `backend/app/services/tool_executors/blog_tool_executor.py`
- `backend/app/services/ai_agents/blog_*`

## Scope
- Create `backend/app/studio/documents/blog/`.
- Move blog route-facing service, generation logic, models, tool specs, and tests into that one directory or its immediate neighbors.
- Expose a small public surface from the blog slice and update route imports to use it.
- Use this ticket to prove the vertical-slice pattern before migrating other studio items.

## Out of scope
- Migrating other document types.
- Behavioral redesign of blog generation.
- Abstracting shared code prematurely.

## Deliverables
- Blog slice under `studio/documents/blog/`.
- Updated route imports.
- A migration pattern others can copy.

## Acceptance criteria
- A maintainer can understand the blog feature by opening one directory tree.
- Blog-specific code no longer lives in generic top-level executor/tool buckets.
- The route continues to work through the new public surface.

## Notes
This is the pattern-setting ticket. Keep the slice deep and self-explanatory.
