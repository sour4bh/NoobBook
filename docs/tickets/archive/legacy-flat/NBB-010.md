## Goal
Finish the document category after the blog pilot proves the shape.

## Current repo cues
- `backend/app/api/studio/business_reports.py`
- `backend/app/api/studio/prds.py`
- `backend/app/api/studio/presentations.py`
- `backend/app/services/tool_executors/business_report_*`
- `backend/app/services/tool_executors/prd_*`
- `backend/app/services/tool_executors/presentation_*`

## Scope
- Create or complete `studio/documents/business_report/`, `studio/documents/prd/`, and `studio/documents/presentation/`.
- Move each item's route-facing service, generation logic, models, and prompt/tool assets into the owning slice.
- Create `studio/documents/shared/` only if there is real shared logic deeper than forwarding wrappers.
- Update route imports to use the new public surfaces.

## Out of scope
- Marketing emails (move them in the marketing ticket if they are marketing-owned).
- Cross-category refactors.
- Behavioral rewrites of generation logic.

## Deliverables
- Owned document slices for business reports, PRDs, and presentations.
- Updated imports and tests.
- Minimal document-shared helpers only where justified.

## Acceptance criteria
- Document items are co-located under `studio/documents/`.
- No document feature depends on a generic executor bucket for ownership.
- Shared code exists only where it actually reduces cognitive load.

## Notes
Do not invent a shared layer just because three items exist. Duplicate until the deeper abstraction is obvious.
