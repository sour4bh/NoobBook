## Goal
Finish the remaining studio categories without inventing new convenience buckets.

## Current repo cues
- `backend/app/api/studio/flash_cards.py`
- `backend/app/api/studio/mind_maps.py`
- `backend/app/api/studio/quizzes.py`
- `backend/app/api/studio/audio.py`
- `backend/app/api/studio/videos.py`

## Scope
- Create or complete `studio/learning/flash_card/`, `mind_map/`, and `quiz/`.
- Create or complete `studio/media/audio/` and `video/` (or `audio_video/` if that is the chosen category name).
- Move route-facing services, generation logic, models, and assets into the owning slices.
- Keep category-shared code minimal and explicit.

## Out of scope
- Document, marketing, or design items.
- Prompt redesigns.
- Global abstractions across all studio categories.

## Deliverables
- Learning item slices.
- Media item slices.
- Updated imports and tests.

## Acceptance criteria
- Learning and media items are owned by their category paths.
- No learning/media feature depends on generic executor ownership.
- The final studio tree reads like the product surface, not the implementation mechanism.

## Notes
Prefer category names that will still make sense when new items are added later.
