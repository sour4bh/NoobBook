## Goal
Keep the React/Vite shell, but make semantic ownership obvious immediately underneath it.

## Current repo cues
- `frontend/src/components/`
- `frontend/src/components/hooks/useVoiceRecording.ts`
- `frontend/src/hooks/use-mobile.tsx`
- `frontend/src/hooks/useAuth.tsx`
- `frontend/src/contexts/PermissionsContext.tsx`

## Scope
- Keep `components/`, `hooks/`, `lib/`, and `contexts/` as allowed top-level shells.
- Move `frontend/src/components/hooks/useVoiceRecording.ts` to `frontend/src/components/chat/hooks/useVoiceRecording.ts`.
- Remove the generic `components/hooks/` bucket once the move is done.
- Keep `hooks/use-mobile.tsx` as a shared hook.
- Keep `hooks/useAuth.tsx` only if it is the thin, app-wide public auth hook; otherwise move feature-owned auth behavior under auth ownership.
- Move any feature-local providers or hooks under the owning domain subtree rather than leaving them in global shells.

## Out of scope
- Inventing a new `features/` taxonomy.
- Large UI rewrites.
- Backend changes.

## Deliverables
- Updated frontend placement guidance.
- Moved feature-owned hooks/providers.
- Deleted `components/hooks/` bucket.

## Acceptance criteria
- No feature-owned hooks remain in `frontend/src/components/hooks/`.
- Root `hooks/` holds only truly shared hooks.
- Frontend contributors can still navigate the repo using conventional React/Vite expectations.

## Notes
This is deliberately not a war on framework conventions. The rule is: keep the shell, tighten the meaning.
