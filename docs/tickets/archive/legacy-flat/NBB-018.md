## Goal
Finish the migration by making the new structure the only structure.

## Current repo cues
- `backend/tests/`
- `frontend/src/components/hooks/`
- `backend/app/services/`
- `backend/app/utils/`

## Scope
- Reorganize backend tests so they mirror the new domain/platform structure.
- Update frontend tests similarly if present.
- Remove temporary import shims introduced during the migration.
- Delete or fully retire legacy directories such as `backend/app/services/`, `backend/app/utils/`, and `frontend/src/components/hooks/` once imports are migrated.
- Keep or expand the structural check so the repo does not regress into convenience buckets.

## Out of scope
- Net-new product features.
- Major behavior changes unrelated to the migration.
- Security or architectural redesign tickets outside file ownership cleanup.

## Deliverables
- Tests organized by owned domains.
- Removed shims and legacy directories.
- Long-term structural enforcement.

## Acceptance criteria
- No runtime imports depend on legacy paths.
- Tests mirror the new ownership structure closely enough to reinforce navigation.
- Structural guardrails remain in place after cleanup.

## Notes
Do this last. Cleanup too early will make the migration more painful than it needs to be.
