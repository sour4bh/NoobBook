# NBB-001 - Foundation and Migration Guardrails

## Goal

Stop new structural drift, reconcile stale repository guidance, and put a real safety net in place before domain migrations begin.

## Outcome

Contributors can find the active repo/branch guidance, new code has approved destinations, CI exists, and baseline route/auth tests catch obvious breakage before the migration moves core behavior.

## Child Tickets

<a id="nbb-101"></a>
### NBB-101 - Reconcile repo identity, default branch, remotes, and contribution target

**Goal:** Remove stale branch and repository instructions before contributors begin migration work.

**Current repo cues:**
- Local branch is `main`.
- `origin` points to `sour4bh/NoobBook`.
- `CONTRIBUTING.md` points at `amitdevv/NoobBook.git` and says all work targets `develop`.

**Scope:**
- Decide and document the canonical repo, default branch, and PR target.
- Update contributor-facing docs that mention stale clone URLs or branch rules.
- Keep the result factual and short; do not redesign the contribution workflow beyond removing contradictions.

**Out of scope:**
- Branch renames.
- GitHub repository settings changes.

**Acceptance criteria:**
- New docs do not say `develop only` unless that branch is explicitly restored as the active target.
- New docs do not reference `amitdevv/NoobBook.git` as the clone URL.
- The contribution target is discoverable before any migration ticket starts.

**Verification:**
- `rg "develop|amitdevv/NoobBook" docs/tickets AGENTS.md CLAUDE.md CONTRIBUTING.md REFACTORING.md`

**Traceability:** Covers the stale branch/repo finding.

### NBB-103 - Bootstrap GitHub Actions CI and add early no-new-legacy-files guardrail

**Goal:** Create a CI lane and block new files in frozen legacy buckets before migration starts.

**Current repo cues:**
- There is no `.github/workflows/` directory.
- Current tickets assumed CI would exist.

**Scope:**
- Use GitHub Actions as the CI platform.
- Add a minimal workflow that runs backend tests and the structural guardrail.
- Add a structural check that fails on new files under frozen destinations except allowlisted temporary shims.
- Frozen destinations must include `backend/app/services/`, `backend/app/utils/`, `backend/app/services/ai_agents/`, `backend/app/services/ai_services/`, `backend/app/services/tool_executors/`, `backend/app/services/tools/`, `backend/app/services/studio_services/jobs/`, `backend/app/services/studio_services/studio_processing/`, and `frontend/src/components/hooks/`.

**Out of scope:**
- Rich import-boundary enforcement; that belongs to `NBB-704`.
- Full frontend CI unless `NBB-108` chooses to add it.

**Acceptance criteria:**
- CI exists in GitHub Actions.
- CI fails when new non-allowlisted files are added to frozen legacy buckets.
- The allowlist is explicit and reviewed.

**Verification:**
- Run the structural check locally.
- Confirm GitHub Actions workflow calls the check.

**Traceability:** Covers missing CI and guardrail timing findings.

### NBB-104 - Create backend skeleton, charters, dependency graph, and route-adapter policy

**Goal:** Create backend destinations and define package rules before moving behavior.

**Scope:**
- Create backend package skeletons for the approved domain roots.
- Define a dependency graph: domains may depend on platform and base; route adapters call domain public surfaces; domains must not reach into each other's internals.
- Define `backend/app/api` as transport-only for this program.
- Define narrow charters for `base/` and every `shared/` directory family. No later ticket may create `base/` or `shared/` without linking back to this charter.

**Out of scope:**
- Moving route modules into domains; route movement is deferred.
- Large behavior refactors.

**Acceptance criteria:**
- Every new backend root has a stated owner and allowed import direction.
- `base/` has a narrow charter and is not a new `utils/`.
- `shared/` is allowed only when its charter names the domain boundary it serves.
- Route files are documented as transport adapters only.

**Verification:**
- Review package docs and dependency graph.
- `rg "shared/|base/" docs/tickets/epics docs/tickets/README.md`

**Traceability:** Covers `base/`/`shared/` loopholes, route policy, and dependency-direction gaps.
