# NBB-004 - Sources Domain Migration

## Goal

Move source ingestion, importers, citations, stores, and analysis into source-owned homes without moving unsafe analysis code before it is quarantined.

## Unlock Requirements

Source skeleton and ownership-map work may start after `NBB-104` and `NBB-206`. Behavior moves wait for their exact hard gates in `tickets.csv`, including route smokes, contracts, raw-code quarantine, loader support, and tool policy where applicable.

<a id="nbb-401"></a>
### NBB-401 - Create source pipeline skeleton and file-format ownership map

**Goal:** Create pipeline-shaped source destinations and avoid generic file-provider drift.

**Primary write scope:**
- `backend/app/sources/`
- `backend/app/providers/`
- source package README/charters
- source ownership docs

**Scope:**
- Create a source pipeline skeleton for upload, ingestion, extraction, chunking, indexing, citations, and analysis.
- File-format operations that express source behavior live under sources.
- Low-level external API/client calls stay under providers; configured external capabilities stay under connectors.

**Decision map:**

| Current code/concept | Target owner under `backend/app/` | Locked decision |
|---|---|---|
| `utils/file_utils.py` | `sources/file_contract.py` | Owns allowed extension, source category, MIME, and upload size contract. Frontend mirrors this through `NBB-205`. Locked target: not `backend/app/api/`, not `platform/files/`, not `providers/files/`. |
| `utils/citation_utils.py` | `sources/citations.py` | Source chunk/citation lookup. Preserve `[[cite:chunk_id]]`; not chat-owned. Verified importer is `backend/app/api/sources/content.py`; docstring-only mentions in text utilities do not change ownership. |
| `utils/source_content_utils.py` | `sources/content.py` | Source-content read helper used by studio/agent code. |
| `utils/pdf_utils.py` | `sources/pdf/ops.py` | PDF page count and page-byte operations for source extraction. |
| `utils/pptx_utils.py` | `sources/pptx/ops.py` | PPTX conversion operations for source extraction. |
| `utils/docx_utils.py` | `sources/docx/ops.py` | DOCX text extraction for source ingestion. |
| `ai_services/pdf_service.py` | `sources/pdf/extract.py` | PDF extraction behavior. |
| `ai_services/pptx_service.py` | `sources/pptx/extract.py` | PPTX extraction behavior. |
| `ai_services/image_service.py` | `sources/image/extract.py` | Source image extraction behavior; model/client calls stay behind providers/connectors. |
| Link/URL source orchestration | `sources/link/` | URL ingestion and extraction behavior. Vendor search/fetch clients stay outside sources. |
| YouTube source orchestration | `sources/youtube/` | Source-side transcript orchestration. Vendor/library client code stays under provider ownership. |
| Audio source orchestration | `sources/audio/` | Source-side transcription flow. ElevenLabs client code stays under provider ownership. |
| Deep research source processing | `sources/analysis/research/` | Source analysis slice; implementation move happens in `NBB-403`. |
| `presentation_export_utils.py` | Not `NBB-401`; `NBB-502`/`NBB-705D` | Studio export ownership. |
| `screenshot_utils.py` | Not `NBB-401`; `NBB-502`/`NBB-705D` | Studio screenshot/export ownership. |
| `path_utils.py` | Not `NBB-401`; approved exception unless `NBB-705E` rehomes it | Cross-cutting filesystem path helper. |
| `logger.py`, `utils/text/` | Not `NBB-401`; approved exceptions unless `NBB-705E` rehomes them | Bootstrap logging and cohesive text package. |

- `platform/files/` and `providers/files/` are not targets for any row in this map.
- Any newly discovered source file-format helper must be added to this map before it moves.

**Out of scope:**
- Moving implementations.
- Changing extraction behavior.

**Acceptance criteria:**
- Source pipeline has obvious homes.
- The decision map above is copied into the created source README/charter.
- New source code has a destination before moves begin.

**Verification:** Review the decision map against current processors and utilities.

**Traceability:** Covers generic file-adapter target contradiction.

### NBB-402 - Move ingestion, importers, source stores, citations, and source-content helpers

**Goal:** Give source ingestion and citations one obvious domain home.

**Primary write scope:**
- `backend/app/services/source_services/`
- `backend/app/sources/`
- `backend/app/utils/file_utils.py`
- `backend/app/utils/citation_utils.py`
- `backend/app/utils/source_content_utils.py`
- source route smoke/contract tests

**Scope:**
- Move source ingestion orchestration, importer-specific logic, source stores, citation helpers, source-content helpers, and source status handling into sources.
- Apply these target paths:
  - `source_services/source_service.py` -> `sources/catalog.py`.
  - `source_services/source_upload/file_upload.py` -> `sources/upload/file.py`.
  - `source_services/source_processing/source_processing_service.py` -> `sources/pipeline.py`.
  - `source_services/source_index_service.py` -> `sources/index.py`.
  - `utils/file_utils.py` -> `sources/file_contract.py`.
  - `utils/citation_utils.py` -> `sources/citations.py`.
  - `utils/source_content_utils.py` -> `sources/content.py`.
- Coordinate any discovered source-adjacent store movement with `NBB-204`; there is no current `NBB-209A` through `NBB-209E` source-store slice.
- Preserve citation marker/lookup contract from `NBB-205`.
- Use refactory: the 7-row target map above populates `docs/tickets/move-plan.csv` with `refactory:move_module` rows plus `refactory:rename_symbol` rows wherever the moved module's public class changes name. Append each row with `ticket=NBB-402`.
- Forbidden filename tokens in the migrated source tree: `service.py`, `manager.py`, `processor.py`, `handler.py`, `helper.py`, and `util.py`. Forbidden vague verb fragments in new source filenames: `handle`, `process`, `manage`, `execute`, `perform`, and `do`. `sources/pipeline.py`, `sources/catalog.py`, `sources/upload/file.py`, and `sources/index.py` are the locked names for this ticket.

**Out of scope:**
- Structured analysis migration.
- Route module movement.

**Acceptance criteria:**
- Source behavior is not stranded in `services/` or `utils/`.
- Route adapters import source public surfaces.
- Citation contract remains compatible with frontend parsing.
- `rg -l 'sources/.*(service|manager|processor|handler|helper|util)\\.py$' backend/app` returns zero after the move, excluding historical docs/tests.

**Verification:** Route smoke tests and later source/citation tests in `NBB-702`.

**Traceability:** Covers source/citation ownership and cross-stack citation contract.

### NBB-403 - Consolidate CSV, database, Freshdesk, and deep-research analysis slices

**Goal:** Stop scattering analysis features across agents, executors, and tool JSON.

**Primary write scope:**
- `backend/app/services/ai_agents/`
- `backend/app/services/tool_executors/`
- `backend/app/services/tools/`
- `backend/app/sources/analysis/`
- analysis tests

**Scope:**
- Move CSV, database, Freshdesk, and deep-research analysis into `sources/analysis/<feature>` slices.
- Preserve sync tool dispatch versus background runner responsibilities.
- Prompt JSON moves only after `NBB-207A` and `NBB-207B`; tool JSON moves only after `NBB-207A` and `NBB-207C`.
- Claude-visible analysis tools must be classified through `NBB-202B` before exposure.
- Raw-code CSV analysis must already be disabled or flag-off per `NBB-203`.
- File-level moves into `sources/analysis/<feature>/` use `refactory:move_module`; append rows to `docs/tickets/move-plan.csv` with `ticket=NBB-403`. Slice-boundary decisions (which files belong together in a feature directory) are manual and land in follow-up commits.
- Migrated analysis executors must not keep `execute_tool`. Multi-tool routers use `dispatch(...)`; single-action executors use the domain verb. Locked method targets:
  - `csv_tool_executor.py::execute_tool` -> `sources/analysis/csv/tool.py::analyze(...)`.
  - `analysis_executor.py::execute_tool` -> `sources/analysis/csv/run.py::dispatch(...)` while `D-002` keeps raw-code execution disabled outside the dual dev flags.
  - `database_executor.py::execute_tool` -> `sources/analysis/database/tool.py::query(...)`.
  - `freshdesk_executor.py::execute_tool` -> `sources/analysis/freshdesk/tool.py::fetch(...)`.
  - `deep_research_executor.py::execute_tool` -> `sources/analysis/research/tool.py::research(...)`.

**Out of scope:**
- Permanent raw-code replacement; that is tracked by `DEFERRED.md` `D-002`.
- Security redesign beyond respecting the quarantine.

**Acceptance criteria:**
- Each analysis feature can be understood from one directory.
- Analysis-specific code is no longer primarily owned by mechanism buckets.
- No unsafe raw-code execution is made easier or more hidden by the move.
- `rg 'def execute_tool\\(|\\.execute_tool\\(' backend/app/sources/analysis` returns zero after migration, excluding historical docs/tests.

**Verification:** Analysis tests in `NBB-702`.

**Traceability:** Covers structured analysis and deep-research gaps.
