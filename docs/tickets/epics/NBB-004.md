<a id="nbb-004"></a>
# NBB-004 - Sources Domain Migration

## Goal

Move source ingestion, connectors, citations, stores, and analysis into source-owned homes without moving unsafe analysis code before it is quarantined.

## Unlock Requirements

Sources may start after `NBB-106`, `NBB-203`, `NBB-204`, `NBB-206`, `NBB-207`, and source-relevant slices of `NBB-209`.

## Child Tickets

<a id="nbb-401"></a>
### NBB-401 - Create source pipeline skeleton and file-format ownership map

**Goal:** Create pipeline-shaped source destinations and avoid `platform/files` drift.

**Scope:**
- Create a source pipeline skeleton for upload, ingestion, extraction, chunking, indexing, citations, and analysis.
- Map PDF/PPTX/DOCX/image/audio/link/YouTube/research/source-kind ownership.
- File-format operations that express source behavior live under sources.
- Vendor SDK calls stay under platform.

**Out of scope:**
- Moving implementations.
- Changing extraction behavior.

**Acceptance criteria:**
- Source pipeline has obvious homes.
- The file-format map follows the audit, not generic `platform/files`.
- New source code has a destination before moves begin.

**Verification:** Review source map against current processors and utilities.

**Traceability:** Covers platform/file adapter contradiction.

<a id="nbb-402"></a>
### NBB-402 - Move ingestion, connectors, source stores, citations, and source-content helpers

**Goal:** Give source ingestion and citations one obvious domain home.

**Scope:**
- Move source ingestion orchestration, connector-specific logic, source stores, citation helpers, source-content helpers, and source status handling into sources.
- Coordinate store moves with `NBB-209`.
- Preserve citation marker/lookup contract from `NBB-205`.

**Out of scope:**
- Structured analysis migration.
- Route module movement.

**Acceptance criteria:**
- Source behavior is not stranded in `services/` or `utils/`.
- Route adapters import source public surfaces.
- Citation contract remains compatible with frontend parsing.

**Verification:** Route smoke tests and later source/citation tests in `NBB-702`.

**Traceability:** Covers source/citation ownership and cross-stack citation contract.

<a id="nbb-403"></a>
### NBB-403 - Consolidate CSV, database, Freshdesk, and deep-research analysis slices

**Goal:** Stop scattering analysis features across agents, executors, and tool JSON.

**Scope:**
- Move CSV, database, Freshdesk, and deep-research analysis into `sources/analysis/<feature>` slices.
- Preserve sync tool dispatch versus background runner responsibilities.
- Prompt/tool JSON moves only after `NBB-207`.
- Raw-code CSV analysis must already be disabled or flag-off per `NBB-203`.

**Out of scope:**
- Permanent raw-code replacement; that is `NBB-404`.
- Security redesign beyond respecting the quarantine.

**Acceptance criteria:**
- Each analysis feature can be understood from one directory.
- Analysis-specific code is no longer primarily owned by mechanism buckets.
- No unsafe raw-code execution is made easier or more hidden by the move.

**Verification:** Analysis tests in `NBB-702`.

**Traceability:** Covers structured analysis and deep-research gaps.

<a id="nbb-404"></a>
### NBB-404 - Replace raw-code analysis with declarative analysis or permanent sandbox

**Goal:** Replace the temporary raw-code quarantine with a durable safe boundary.

**Scope:**
- Replace model-written Python execution with a declarative analysis plan, or build a permanent sandbox boundary with explicit resource/capability limits.
- Update tool schema, prompts, tests, and documentation to match the chosen permanent boundary.
- Remove the temporary flag-off state only when the permanent boundary is proven.

**Out of scope:**
- Moving source analysis files; that is `NBB-403`.

**Acceptance criteria:**
- Production analysis no longer depends on unrestricted free-form Python execution.
- The boundary is explicit, testable, and documented.
- The temporary mitigation from `NBB-203` is removed or reduced to a defense-in-depth guard.

**Verification:** Security and analysis contract tests.

**Traceability:** Covers permanent raw-code analysis hardening.
