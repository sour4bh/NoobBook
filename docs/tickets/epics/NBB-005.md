# NBB-005 - Studio Domain Migration

## Goal

Move studio into category-owned slices while preserving service, job, tool, and run layers where they exist.

## Unlock Requirements

Studio may start after `NBB-106`, `NBB-204`, `NBB-205`, `NBB-206`, `NBB-207`, and `NBB-210`.

## Child Tickets

<a id="nbb-501"></a>
### NBB-501 - Finalize canonical studio taxonomy and item registry

**Goal:** Choose studio categories and item ownership before item migrations begin.

**Scope:**
- Inventory all studio routes/items.
- Choose canonical categories and item directory names.
- Produce a registry mapping current route names, services, jobs, tool executors, prompt/tool assets, tests, and target item paths.
- Avoid placeholders such as "or chosen later".

**Out of scope:**
- Moving implementations.

**Acceptance criteria:**
- Every studio item has one target category/item path.
- Blog, business reports, flow diagrams, and other ambiguous items have explicit ownership.
- New studio items know where to land.

**Verification:** Compare registry to `backend/app/api/studio`, `studio_services`, `tool_executors`, tools, and prompts.

**Traceability:** Covers studio taxonomy ambiguity.

<a id="nbb-502"></a>
### NBB-502 - Map studio service/job/tool/run layers and status/progress contracts

**Goal:** Document the four-layer reality before moving individual studio features.

**Scope:**
- For every studio item, map route adapter, domain service, job/background layer, sync tool dispatch, background run/agent orchestration, prompt/tool JSON, output storage, and tests.
- Preserve the distinction between sync tool dispatch and background orchestration.
- Tie studio job status/progress/result contracts to `NBB-205` and background ownership to `NBB-210`.

**Out of scope:**
- Merging layers for convenience.

**Acceptance criteria:**
- Each studio item has a service/job/tool/run map.
- Migrators cannot silently merge jobs into executors.
- Shared studio behavior has a chartered home.

**Verification:** Review map against `studio_services/jobs`, `tool_executors`, and studio routes.

**Traceability:** Covers studio jobs and executor-pair invariant.

<a id="nbb-503"></a>
### NBB-503 - Pilot one registry-selected studio item

**Goal:** Validate the studio map before migrating all categories.

**Scope:**
- Pick one item from the finalized registry.
- Move that item as a complete vertical slice.
- If the pilot proves `NBB-501` or `NBB-502` wrong, revise those decisions before `NBB-504` through `NBB-507`.

**Out of scope:**
- Bulk studio migration.

**Acceptance criteria:**
- Pilot demonstrates the final file pattern.
- Pilot may feed corrections back into taxonomy/layer decisions.
- No follow-on studio migration starts until pilot lessons are applied.

**Verification:** Studio route smoke tests and item-specific behavior checks.

**Traceability:** Covers pilot inversion ambiguity.

<a id="nbb-504"></a>
### NBB-504 - Migrate document items

**Goal:** Move document generators into registry-approved document homes.

**Scope:**
- Migrate document-category items from the registry.
- Co-locate item-specific service/job/tool/run/prompt/tool-schema/tests with each item.
- Use shared code only through chartered `shared/` homes.

**Out of scope:**
- Marketing, design, learning, or media items.

**Acceptance criteria:**
- Document items no longer depend on generic executor ownership.
- Prompt/tool JSON movement follows `NBB-207`.
- Route adapters use item public surfaces.

**Verification:** Studio smoke tests and later `NBB-703`.

**Traceability:** Covers studio document migration.

<a id="nbb-505"></a>
### NBB-505 - Migrate marketing items

**Goal:** Move marketing generators into registry-approved marketing homes.

**Scope:** Same migration pattern as `NBB-504`, applied to registry-approved marketing items.

**Acceptance criteria:**
- Marketing item ownership is visible from paths.
- No unchartered `shared/` helper bucket appears.

**Verification:** Studio smoke tests and later `NBB-703`.

**Traceability:** Covers studio marketing migration.

<a id="nbb-506"></a>
### NBB-506 - Migrate design items and logo/brand support

**Goal:** Move design generators and settle logo/brand support ownership.

**Scope:**
- Migrate registry-approved design items.
- Resolve `logo_utils` ownership against brand/design registry decisions.
- Keep platform/image/file helpers out of studio unless they are studio policy.

**Acceptance criteria:**
- Logo/brand support ownership is explicit.
- Design item paths are predictable.

**Verification:** Studio smoke tests and later `NBB-703`.

**Traceability:** Covers logo/brand ambiguity.

<a id="nbb-507"></a>
### NBB-507 - Migrate learning media items

**Goal:** Move learning and media generators into registry-approved homes.

**Scope:** Migrate flash cards, mind maps, quizzes, audio, video, and any registry-approved learning/media item.

**Acceptance criteria:**
- Learning/media category names are final, not placeholders.
- Item-specific code is co-located.

**Verification:** Studio smoke tests and later `NBB-703`.

**Traceability:** Covers remaining studio categories.
