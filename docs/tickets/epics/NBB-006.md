# NBB-006 - Frontend Ownership Tightening

## Goal

Keep the React/Vite shell while making ownership truthful under that shell.

## Child Tickets

<a id="nbb-601"></a>
### NBB-601 - Preserve frontend shells while tightening placement rules

**Goal:** Document what root frontend shells mean and where feature code belongs.

**Scope:**
- Keep `components`, `hooks`, `lib`, and `contexts` as conventional shells.
- Define their allowed contents.
- Coordinate with `NBB-108` for frontend test decision.

**Acceptance criteria:**
- Root shells remain, but each has a strict meaning.
- `components/ui` remains design-system primitives.

**Verification:** Review frontend placement docs.

**Traceability:** Covers frontend being underplanned.

<a id="nbb-602"></a>
### NBB-602 - Move feature-owned hooks providers under owning domains

**Goal:** Stop feature hooks and providers from drifting into root buckets.

**Scope:**
- Move feature-owned hooks/providers into chat, sources, studio, auth, settings, or other owning frontend domains.
- Keep root hooks only for truly app-wide hooks.
- Keep contexts only for app-wide providers.

**Acceptance criteria:**
- `frontend/src/components/hooks` is deleted or legacy-only.
- Feature-local hooks/providers have owning homes.

**Verification:** Search for feature hooks in root buckets.

**Traceability:** Covers `useVoiceRecording`, `useAuth`, and contexts concerns.

<a id="nbb-603"></a>
### NBB-603 - Tighten lib contexts API clients citations exports and logger ownership

**Goal:** Keep `lib` and `contexts` truthful without fighting React conventions.

**Scope:**
- Keep API/browser/auth adapters in `lib` when they are real boundaries.
- Review citations, export helpers, logger, and generic utilities for domain ownership.
- Coordinate citation and contract changes with `NBB-205`.

**Acceptance criteria:**
- Product behavior is not parked in `lib` by convenience.
- Citation frontend behavior remains compatible with backend contract.

**Verification:** Search remaining `lib` files for product nouns and justify each.

**Traceability:** Covers frontend lib/contexts and citation contract.

<a id="nbb-604"></a>
### NBB-604 - Normalize frontend domain subtrees and design-system guardrails

**Goal:** Make chat, sources, and studio frontend subtrees consistently navigable.

**Scope:**
- Establish local patterns for hooks, types, helpers, and tests inside major frontend domains.
- Preserve Phosphor/shadcn/design-system rules from repo guidance.
- Do not invent a new top-level `features/` taxonomy.

**Acceptance criteria:**
- Major frontend domains have consistent local structure.
- Shared UI remains in `components/ui`.

**Verification:** Review frontend tree and design-system docs.

**Traceability:** Covers frontend drift.
