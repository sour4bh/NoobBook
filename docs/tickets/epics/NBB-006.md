# NBB-006 - Frontend Ownership Tightening

## Goal

Keep the React/Vite shell while making ownership truthful under that shell.

## Unlock Requirements

Frontend ownership work may start after `NBB-105` and the machine-visible frontend test decision in `NBB-108A`. If `NBB-108A` chooses to implement tests, `NBB-108B` may run in parallel with ownership moves.

<a id="nbb-601"></a>
### NBB-601 - Preserve frontend shells while tightening placement rules

**Goal:** Document what root frontend shells mean and where feature code belongs.

**Primary write scope:**
- `frontend/src/components/`
- `frontend/src/hooks/`
- `frontend/src/lib/`
- `frontend/src/contexts/`
- frontend placement docs

**Scope:**
- Keep `components`, `hooks`, `lib`, and `contexts` as conventional shells.
- Define their allowed contents.
- Coordinate with `NBB-108A` for frontend test decision.

**Out of scope:**
- Bulk moving frontend files.
- Design-system rewrites.

**Acceptance criteria:**
- Root shells remain, but each has a strict meaning.
- `components/ui` remains design-system primitives.

**Verification:** Review frontend placement docs.

**Traceability:** Covers frontend being underplanned.

<a id="nbb-602"></a>
### NBB-602 - Move feature-owned hooks providers under owning domains

**Goal:** Stop feature hooks and providers from drifting into root buckets.

**Primary write scope:**
- `frontend/src/components/hooks/`
- `frontend/src/components/chat/`
- `frontend/src/components/sources/`
- `frontend/src/components/studio/`
- `frontend/src/hooks/`
- `frontend/src/contexts/`

**Scope:**
- Move feature-owned hooks/providers into chat, sources, studio, auth, settings, or other owning frontend domains.
- Keep root hooks only for truly app-wide hooks.
- Keep contexts only for app-wide providers.
- This ticket depends on the `NBB-108A` decision. It does not wait for `NBB-108B` unless `NBB-108A` explicitly changes the frontend gate.
- Frontend moves and renames use `refactory:move_module` / `refactory:rename_symbol`; refactory's ts-morph backend handles `.tsx`/`.ts` path aliases and barrel re-exports. Append one row per operation to `docs/tickets/move-plan.csv` with `language=typescript` and `ticket=NBB-602`.

**Out of scope:**
- Renaming app-wide hooks.
- React/Vite shell redesign.

**Acceptance criteria:**
- `frontend/src/components/hooks` is deleted or legacy-only.
- Feature-local hooks/providers have owning homes.

**Verification:** Search for feature hooks in root buckets.

**Traceability:** Covers `useVoiceRecording`, `useAuth`, and contexts concerns.
