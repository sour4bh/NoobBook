# NBB-008 - Services Eradication and Final Boundary Enforcement

## Goal

Finish the strict post-sprint migration that NBB-706 deliberately stopped short
of: remove every tracked file under `backend/app/services/`, migrate all live
behavior into truthful domain, provider, or connector homes, and enforce that
`app.services.*` cannot return.

NBB-706 is considered sprint-complete under the approved-exception standard.
This epic supersedes those placement exceptions. It does not reopen HTTP
contracts, auth semantics, storage schemas, or deferred route movement unless a
services move exposes a direct regression.

## Execution Constraints

- No compatibility shim under `app.services` may be introduced.
- Mechanical moves should use Refactory when it is safe and useful. Refactory
  project root must be the absolute Python package root (`<worktree>/backend`),
  preview first, then `apply: true`.
- Every moved module or JSON asset appends a row to `docs/tickets/move-plan.csv`.
- Each ticket must run the architecture, AST, project-id, DAG, backend pytest,
  and pyright checks listed in the verification section unless it records a
  concrete pre-existing blocker.
- If implementation discovers a missing owner, patch this epic before moving
  product code.
