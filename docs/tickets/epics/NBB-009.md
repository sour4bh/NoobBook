# NBB-009 - Deferred Closure and Runtime Hardening

<a id="nbb-009"></a>
## Epic Goal

Close the deferred post-migration items that are now small enough to finish:
config/runtime cleanup, auth transport parity, OAuth callback state validation,
project access semantics, API transport-boundary enforcement, safe CSV analysis,
and frontend smoke coverage.

`D-003` (broad security review) and `D-005` (cross-stack contract redesign)
remain deferred by explicit decision. This epic records that boundary instead
of silently expanding into a new migration.

## Execution Notes

- Run mostly serially because config, app bootstrap, auth middleware, and
  source-analysis contracts share import and behavior surfaces.
- `NBB-908` may run after `NBB-901`; it is frontend-only.
- No compatibility shims are allowed for old config module names.
- HTTP route paths stay stable.

### NBB-902 - Rebuild config as a Hydra-like Pydantic runtime tree

**Goal:** Make runtime configuration a real `app.config` subtree and remove the
old top-level module collision.

**Primary write scope:**
- `backend/requirements.txt`
- `backend/app/config/`
- `backend/app/__init__.py`
- config imports across `backend/app/` and backend tests
- current docs that describe config module paths

**Decision map:**

| Old path | New path |
|---|---|
| `backend/config.py` | `backend/app/config/runtime.py` |
| `backend/app/config/asset_registry.py` | `backend/app/config/asset.py` |
| `backend/app/config/brand_context_loader.py` | `backend/app/config/brand.py` |
| `backend/app/config/context_loader.py` | `backend/app/config/context.py` |
| `backend/app/config/model_loader.py` | `backend/app/config/model.py` |
| `backend/app/config/prompt_loader.py` | `backend/app/config/prompt.py` |
| `backend/app/config/tier_loader.py` | `backend/app/config/provider.py` |
| `backend/app/config/tool_loader.py` | `backend/app/config/tool.py` |

**Scope:**
- Add `pydantic-settings`.
- Keep `backend/app/settings/env.py` and `backend/app/settings/validation.py`
  as the user-facing settings mutation surface.
- Preserve `.env` loading and existing `Config` class callers through the new
  runtime module.

**Out of scope:** Replacing the settings UI write path.

**Acceptance criteria:** No tracked `backend/config.py` remains and imports use
`app.config.runtime`, `app.config.provider`, `app.config.prompt`,
`app.config.tool`, `app.config.model`, `app.config.asset`, `app.config.context`,
or `app.config.brand`.

**Verification:** Runtime config tests, app-factory import smoke, full backend

### NBB-904 - Fix OAuth callback auth boundary

**Goal:** Allow provider OAuth redirects to complete without a bearer token
while validating a signed state token.

**Primary write scope:**
- `backend/app/__init__.py`
- `backend/app/api/__init__.py`
- `backend/app/api/google/oauth.py`
- `backend/app/providers/google/auth.py`
- auth/OAuth tests

**Scope:**
- `/api/v1/google/callback` bypasses the generic JWT guard.
- `/api/v1/google/auth` emits an opaque signed state token generated from the
  authenticated request identity.
- The callback rejects missing, invalid, or expired state before token exchange.

**Out of scope:** DB-backed nonce storage and broader OAuth-provider redesign.

**Acceptance criteria:** Callback without bearer auth does not 401; invalid
state fails closed; valid signed state reaches token exchange with the original
user id.

**Verification:** OAuth callback tests plus common backend checks.
